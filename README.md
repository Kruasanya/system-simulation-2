# system-simulation-2
## Что реализовано

В работе реализована модифицированная модель Басса для двух конкурирующих компаний.
Рынок разделен на потенциальных клиентов, а также на клиентов компаний A и B.
Клиенты каждой компании делятся на три группы:

- довольные;
- нейтральные;
- недовольные.

Довольные клиенты участвуют в механизме сарафанного радио.
Нейтральные и недовольные клиенты могут быть переманены конкурентом.
Недовольные клиенты дополнительно могут возвращаться в пул потенциальных клиентов.

## Структура модели

Для компании A поток новых клиентов задается как:

`adopt_A = pA * potential + qA * A_satisfied * potential / N`

Аналогично задается поток для компании B.

Переманивание моделируется через взаимодействие довольных клиентов одной компании
с уязвимыми клиентами конкурента:

`switch_to_A = aggression * tolerance * A_satisfied * (B_neutral + B_dissatisfied) / N`

`switch_to_B = aggression * tolerance * B_satisfied * (A_neutral + A_dissatisfied) / N`

Возврат недовольных клиентов на рынок:

`return_A = disappointment * A_dissatisfied`

`return_B = disappointment * B_dissatisfied`

После каждого притока новые клиенты делятся в пропорции:

- 50% довольные;
- 30% нейтральные;
- 20% недовольные.

## Оценка сходимости

Модель запускалась на горизонте 200 шагов.
В качестве критерия сходимости используется максимальное отклонение доли рынка компании A
на последних 20 шагах:

`max |share_A(t) - mean(share_A_tail)|`

Если это отклонение меньше `1e-3`, динамика считается сошедшейся.

Для базового сценария были получены следующие результаты:

- итоговая доля компании A: около `0.5911`;
- итоговая доля компании B: около `0.4089`;
- максимальное отклонение на хвосте: около `2.429e-05`;
- сходимость достигнута.

Графики сходимости приложены отдельно в файлах:

- `convergence_shares.png`;
- `convergence_tail.png`.

## Варьирование параметров

Для построения датасета варьировались параметры `pA` и `pB` на равномерной сетке.
Остальные параметры фиксировались:

- `qA = 0.30`;
- `qB = 0.30`;
- `disappointment = 0.02`;
- `tolerance = 0.05`;
- `aggression = 0.10`.

Для каждой пары параметров сохранялись:

- `share_A`;
- `share_B`;
- признак сходимости;
- максимальное отклонение на хвосте.

## ML-модель

Для аппроксимации функции `f(pA, pB) -> share_A` были рассмотрены три полиномиальные регрессии:

- линейная;
- квадратичная;
- кубическая.

Обучение выполнено через метод наименьших квадратов `numpy.linalg.lstsq`.
На тестовой выборке лучшей оказалась кубическая модель.

Проверочный результат на укороченной сетке:

- `MAE = 0.018825`;
- `R2 = 0.979084`.

## Обратная задача

В качестве целевой доли рынка была выбрана величина:

`share_A = 0.60`

Далее находилась область параметров `(pA, pB)`, для которой прогноз модели
отличается от целевого значения не более чем на `7%`.

Эта область визуализирована на графике `inverse_region.png`.
